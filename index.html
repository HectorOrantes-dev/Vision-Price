Machine Learning

1. Descripción de la implementación de Machine Learning

Modelos predictivos y análisis de datos históricos para garantizar presupuestos estables y confiables.

1.1

Estrategia de datos: dataset sintético y API de Mercado Libre

En la primera fase del MVP, el Motor de Materiales se entrena utilizando un conjunto de datos sintético que simula el comportamiento histórico de los precios de materiales de construcción en México. Este dataset considera las fluctuaciones del mercado, incluyendo incrementos de precios y descuentos asociados a eventos comerciales como Hot Sale y Buen Fin, en materiales como cemento, pintura, azulejos y losetas. Los datos se generan a partir de rangos de precios reales y de la volatilidad mensual del sector, la cual puede alcanzar hasta un 18 %. Esta estrategia permite desarrollar, entrenar y validar los modelos de minería de datos desde el primer sprint, sin depender de la disponibilidad de fuentes de datos externas y evitando la necesidad de implementar técnicas de web scraping, las cuales no resultan viables dentro del alcance del proyecto.

En paralelo, un servicio programado (scheduler) consulta cada hora la API oficial de Mercado Libre para obtener los precios actuales de los materiales de construcción asociados a tres tipos de trabajo: remodelación de cocina, construcción nueva y pintura de interiores o exteriores. Los datos reales obtenidos alimentan progresivamente el pipeline de datos y sustituyen al dataset sintético conforme se acumula historial suficiente, de modo que los modelos transitan de un entorno controlado de entrenamiento a datos de producción sin cambios en la arquitectura.

1.2

Técnicas de minería de datos aplicadas (scikit-learn)

Sobre el conjunto de datos (sintético en la fase inicial y real conforme avanza la recolección vía API) se aplican tres técnicas complementarias:

  • 1. Clustering K-Means. Agrupa los productos obtenidos de Mercado Libre por categoría, rango de precio y tipo de trabajo, lo que permite ofrecer al usuario alternativas equivalentes para cada material calculado (opción económica, estándar y premium) y detectar la pertenencia de productos nuevos a familias de materiales ya conocidas.
  • 2. Regresión para predicción de precios (Random Forest / Gradient Boosting). Predice el precio esperado de un material cuando la API no devuelve resultados o el producto no está disponible temporalmente, utilizando el historial acumulado como mecanismo de fallback. De esta forma el presupuesto nunca queda incompleto ni se construye con precios desactualizados.
  • 3. Detección de anomalías (Isolation Forest). Identifica precios atípicos en los resultados de la API —errores de publicación, productos mal categorizados o precios expresados en unidades incorrectas— y los excluye antes de que lleguen al presupuesto del usuario, garantizando estimaciones confiables.
1.3

Integración con la arquitectura

Los precios validados se almacenan en Redis con un TTL de 24 horas para servir consultas inmediatas, y en PostgreSQL como histórico estructurado que sirve de base para el reentrenamiento periódico de los modelos. El servicio de consulta horaria a la API de Mercado Libre se ejecuta como tarea asíncrona dentro del Pricing Service, encolada mediante RabbitMQ, de modo que las actualizaciones de precios no bloquean las solicitudes de los usuarios y pueden reintentarse automáticamente en caso de fallo.

NLP & LLM

2. Descripción de la implementación de Procesamiento de Lenguaje Natural y LLM

Recomendación inteligente en lenguaje natural y automatización en la generación de textos.

2.1

Recomendación inteligente de materiales a partir del escaneo 3D

Una vez que el usuario escanea el espacio con ARCore (Android) o ARKit (iOS) y el Vision Service reconstruye la geometría del lugar (superficies detectadas, dimensiones y tipo de espacio), un modelo de lenguaje (LLM) actúa como capa de recomendación que traduce esos datos técnicos en una propuesta de materiales contextualizada al tipo de obra, expresada en lenguaje natural y comprensible para cualquier perfil de usuario. El flujo de implementación es el siguiente:

  • 1. Construcción del prompt estructurado. El backend integra en un prompt la salida del escaneo 3D (metros cuadrados de muros, pisos y techos), el tipo de trabajo seleccionado (remodelación de cocina, construcción nueva o pintura de interiores/exteriores) y el catálogo de productos vigentes obtenido mediante la API de Mercado Libre.
  • 2. Recomendación razonada de materiales. El LLM genera una propuesta que indica qué productos del catálogo se ajustan mejor al espacio escaneado, en qué cantidades —siempre validadas contra el Motor de Materiales— y por qué. Por ejemplo, sugerir pintura vinílica lavable para una cocina por la humedad del ambiente, o un azulejo de mayor resistencia al tránsito para los pisos.
  • 3. Interacción en lenguaje natural. El usuario puede expresar su necesidad escribiendo, por ejemplo, “quiero remodelar mi cocina de 3×4 con acabados económicos”. El sistema interpreta la intención, extrae los parámetros relevantes (tipo de trabajo, dimensiones, rango de presupuesto) y los mapea a las entidades del sistema sin que el usuario navegue por menús técnicos. Esta capacidad es clave para el perfil de maestro de obra, el usuario con menor experiencia tecnológica.
  • 4. Generación automática de textos del presupuesto. El LLM redacta las descripciones profesionales que acompañan al presupuesto exportado en PDF (alcance del trabajo, notas técnicas y supuestos), elevando la calidad de presentación de la propuesta sin esfuerzo adicional del usuario.
2.2

Integración con la arquitectura

El servicio de inferencia del LLM se expone como un microservicio adicional detrás del API Gateway, consistente con la arquitectura orientada a servicios del proyecto. Las respuestas del modelo se devuelven estructuradas en formato JSON, lo que permite integrarlas de forma directa y verificable con el Motor de Materiales y el Servicio de Precios: las cantidades sugeridas por el LLM nunca sustituyen al cálculo determinístico de las fórmulas por tipo de trabajo, sino que lo complementan con contexto, justificación y lenguaje accesible.

Esta separación de responsabilidades garantiza que el componente de lenguaje natural aporte valor en la experiencia de usuario y en la presentación profesional del presupuesto, mientras que la precisión numérica permanece bajo el control de los algoritmos especializados y de los datos validados por los modelos de machine learning descritos en la sección anterior.

FAQ

Preguntas frecuentes

¿Necesito internet para escanear? +
El escaneo y el cálculo de materiales funcionan sin conexión. Solo necesitas internet para consultar precios en tiempo real de los proveedores. Si no tienes señal en obra, los precios de caché (últimas 24 h) se usan automáticamente.
¿Qué tan precisas son las medidas? +
En condiciones normales de iluminación, la app logra un margen de error menor al 5% respecto a la cinta métrica en superficies rectangulares. Si el resultado no te convence, puedes corregir cualquier medida manualmente antes de calcular materiales.
¿Funciona en iPhone y Android? +
Sí. VisionPrice está disponible para Android (con ARCore, requiere Android 8.0+) e iOS (con ARKit, requiere iPhone 6s o superior con iOS 14+). Una sola cuenta funciona en ambos sistemas.
¿De dónde vienen los precios? +
Los precios reales se obtienen consultando automáticamente cada hora la API oficial de Mercado Libre para tres tipos de trabajo (remodelación, construcción y pintura) y se complementan con un dataset sintético entrenado con volatilidad histórica para cuando los productos no estén disponibles. Esta estrategia asegura que las estimaciones siempre sean confiables y evita por completo el uso de técnicas de web scraping.
¿Puedo cancelar mi suscripción en cualquier momento? +
Sí. Sin penalizaciones ni períodos mínimos. Si cancelas, mantienes el acceso al plan hasta el final del período pagado. Tus proyectos y presupuestos están disponibles para exportar aunque pases al plan gratuito.
¿Mis presupuestos son privados? +
Sí. Solo tú puedes ver tus proyectos. El enlace público que puedes compartir es temporal (30 días), de solo lectura, y lo puedes desactivar en cualquier momento desde la app.
